How to watch video from your computer on every device at home without downloading it[in two steps]

Why ?

I usually watch movies and read books from my android phone and tablet. So I decided, it would be great if I could use my main hard disk and share it to the network. I came across this method accidentally. And it showed itself very useful.

So, You will need

  1. Linux computer. Preferably Debian
  2. Android device
  3. Standart wifi router
  4. A little patience

Let's go on. Step one. Installation.

You will definitely need nginx server. It showed itself pretty decent on even old computers. Install it like this. sudo apt-get install nginx You will probably need ffmpeg file converter. Install it like this sudo apt-get install ffmpeg You will need media player. Install vlc media player on your android device.

Step two. Configuring nginx

Let's have our hands a little bit dirty. Execute next commands sudo -s cd /etc/nginx/sites-enabled touch home-shared Put in file home-shared configuration like this: server { listen 80; location / { autoindex on; root /home/mik/shared; } } Where root is absolute path to your shared folder It would go like this echo "server { listen 80; location / { autoindex on; root /home/mik/shared; } }" > home-shared Restart your nginx server by: /etc/init.d/nginx restart In your browser, go to 127.0.0.1. You must see something like this.

Now, You must try to see this with another computer in your network. Your ip address you can see like this sudo ifconfig | grep "inet addr" Select that one wich starts with 192.168.1. Go to your android device. Or to another computer. And type into address bar that ip. Like this.

If you see directory content. Congratulations. If you don't, try disabling your firewall. Now You can put any video into that directory. And vlc from your android device will play it on the fly.

In the next part of this tutorial I will tell how to convert fast any video, so you can feed it to vlc without any problems.

How to make immutable Record play nicely with typescript

For my current work, I need support for TypeScript with Immutable.js. During my studying. I realised, that Record structure in Immutable.js has named parameters. So. It actually, acts like a JavaScript object. But, with several differences. But,TypeScript does not work good with it. So. I tried to find a solution

Objective

Make immutable.js type Record work with typechecking in TypeScript

Solution

///<reference path='../node_modules/immutable/dist/immutable.d.ts'/> // Declare our interface interface IMeta { editingTagId: number editingNodeRowid: number tagFontMagnifier: number minTagFont: number maxTagFont: number } // Make composite interface interface IMetaComposite extends Immutable.Map,IMeta {} // Make Record var metaRecord = Immutable.Record({ editingTagId: 0, editingNodeRowid: 0, tagFontMagnifier: 1, minTagFont: 14, maxTagFont: 20 }) // Cast composite type var meta = <IMetaComposite>(new metaRecord) // Result. Actually compiles. IDE support works: meta.editingTagId meta.set meta.minTagFont

Conclusion

This is not the best solution. But this is enough for my humble needs.

Naming convention

There are only two hard things in Computer Science: cache invalidation and naming things.

-- Phil Karlton

Let's talk about the second thing. Here is my solution, inspired by tagging and filtering systems. In the future I intend to write a special program for handling this name convention. The core idea is to name functions by separate tags of functionality they are intended to.

Here is a quick overview database_node_withTags(node,tags) database_node_withDescription(node,description) node_reDisplay(*params) node_reDisplay_withFancyGraph(*params) tag_rename display_onEvent_tag conversion_node_tag_tagToNode The trick is to name each element as a separate essence with one restriction. You should name your functions in alphabet order. (You will see later why it is needed)

How is it useful

It answers to a simple question. How can I find something not remembering its name. Here is an example:

tag will find any function that somehow is related to a tag:

  • deletion
  • insertion
  • saving to database
  • deleting from somewhere
  • connecting to node
  • joining events

onEvent.*tag regular expression will find any function that somehow connected to tag AND onEvent at the same time: E.g:

  • saving tag on Event occuring

save.*tag regular expression will find any function that is saving something AND related to a tag

E.g:

  • save tag to database
  • save tag into app state
  • save tag toJson

Your algorithm to implement this convention

  1. Determine delimiter (_ or $ or -) what your language accepts as a name of variable
  2. Determine a number of simple nouns. E.g(is,tag,node,saveToDb,convert,display,e.t.c)
  3. Place this number of nouns in your variable name for
