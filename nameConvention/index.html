Naming convention

There are only two hard things in Computer Science: cache invalidation and naming things.

-- Phil Karlton

Let's talk about the second thing. Here is my solution, inspired by tagging and filtering systems. In the future I intend to write a special program for handling this name convention. The core idea is to name functions by separate tags of functionality they are intended to.

Here is a quick overview database_node_withTags(node,tags) database_node_withDescription(node,description) node_reDisplay(*params) node_reDisplay_withFancyGraph(*params) tag_rename display_onEvent_tag conversion_node_tag_tagToNode The trick is to name each element as a separate essence with one restriction. You should name your functions in alphabet order. (You will see later why it is needed)

How is it useful

It answers to a simple question. How can I find something not remembering its name. Here is an example:

tag will find any function that somehow is related to a tag:

  • deletion
  • insertion
  • saving to database
  • deleting from somewhere
  • connecting to node
  • joining events

onEvent.*tag regular expression will find any function that somehow connected to tag AND onEvent at the same time: E.g:

  • saving tag on Event occuring

save.*tag regular expression will find any function that is saving something AND related to a tag

E.g:

  • save tag to database
  • save tag into app state
  • save tag toJson

Your algorithm to implement this convention

  1. Determine delimiter (_ or $ or -) what your language accepts as a name of variable
  2. Determine a number of simple nouns. E.g(is,tag,node,saveToDb,convert,display,e.t.c)
  3. Place this number of nouns in your variable name for
