Naming convention

There are only two hard things in Computer Science: cache invalidation and naming things.

-- Phil Karlton

Let's talk about the second thing. Here is my solution, inspired by tagging and filtering systems. In the future I intend to write a special program for handling this name convention. The core idea is to name functions by separate tags of functionality they are intended to.

Here is a quick overview

database_node_withTags(node,tags)
database_node_withDescription(node,description)
node_reDisplay(*params)
node_reDisplay_withFancyGraph(*params)
tag_rename
display_onEvent_tag
conversion_node_tag_tagToNode

The trick is to name each element as a separate essence with one restriction. You should name your functions in alphabet order. (You will see later why it is needed)

How is it useful

It answers to a simple question. How can I find something not remembering its name. Here is an example:

tag will find any function that somehow is related to a tag:

  • deletion
  • insertion
  • saving to database
  • deleting from somewhere
  • connecting to node
  • joining events

onEvent.*tag regular expression will find any function that somehow connected to tag AND onEvent at the same time: E.g:

  • saving tag on Event occuring

save.*tag regular expression will find any function that is saving something AND related to a tag

E.g:

  • save tag to database
  • save tag into app state
  • save tag toJson

Your algorithm to implement this convention

  1. Determine delimiter (_ or $ or -) what your language accepts as a name of variable
  2. Determine a number of simple nouns. E.g(is,tag,node,saveToDb,convert,display,e.t.c)
  3. Place this number of nouns in your variable name for

How to make immutable Record play nicely with typescript

For my current work, I need support for TypeScript with Immutable.js. During my studying. I realised, that Record structure in Immutable.js has named parameters. So. It actually, acts like a JavaScript object. But, with several differences. But,TypeScript does not work good with it. So. I tried to find a solution

Objective

Make immutable.js type Record work with typechecking in TypeScript

Solution

///<reference path='../node_modules/immutable/dist/immutable.d.ts'/>
// Declare our interface
interface IMeta {
    editingTagId:     number
    editingNodeRowid: number
    tagFontMagnifier: number
    minTagFont:       number
    maxTagFont:       number
}
// Make composite interface
interface IMetaComposite extends Immutable.Map,IMeta {}
// Make Record
var metaRecord = Immutable.Record({
    editingTagId:     0,
    editingNodeRowid: 0,
    tagFontMagnifier: 1,
    minTagFont:       14,
    maxTagFont:       20
})
// Cast composite type
var meta = <IMetaComposite>(new metaRecord)
// Result. Actually compiles. IDE support works:
meta.editingTagId
meta.set
meta.minTagFont

Conclusion

This is not the best solution. But this is enough for my humble needs.

NW SULFUR is a way to package your node webkit program

Why ?

I needed a way to distribute node-webkit apps in a smaller size package. So. I created nw-sulfur It is still in a very beta stage. But it is useful.

Before nw sulfur:

Your nw folder with binaries is around 150M+

After nw-sulfur:

Your nw-folder is around 20M

It does the following:

  1. Determines if you have to download NW.js
  2. If Yes: Downloads it and Shows fancy progressbar.
  3. If No: Opens your app with NW binary.

How ?

I needed a very small program. So I picked up golang. At first, I wanted to use C. But I found out, that it was quite difficult to make it work on all platforms. I tried to reduce size of resulting binary as much as possible. I also wanted ability to customize progressbar while downloading data. So I took ui based on web browser. Precisely. It works this way:

  1. Start server in a goroutine
  2. Run default platform independent browser.
  3. Do downloading.

Installation and further instructions are here

It is kind of interesting to start to work in a different language and different paradigm. Working in Golang was a pleasure. But. There were some gotchas.

For example: I could not read JSON files. Because Unmarchall structure must start with a capital letter. And there is no clear error about this. Just, cannot Unmarchall, and that is it. It is kind of weird. Thanks to help on stackoverflow. And 2 hours of hitting wall with my head. In the end GO is much more understandable than, node.js, which, in turn, has a lot more WTF on its own.

Launched simple helper for work[pomodorro]

Feautures:

  • keeps track of your sessions, so you can kill it. When you start it will continue working from the same time.
  • has a beautiful timer
  • no dependencies

Install it like this:

git clone https://github.com/VbifRkbvjd/pomodorro-cli

Usage is like this

python ./main.py

Configuration is in config.py

SAVE_FILE = "pomodorro.db"
WORK = 25
REST = 5
BIGREST = 25
ONE_SECOND = 1

If you want to make a pause. Close app by Ctrl+c. It will remember it last position when you start it again. Source goes here
